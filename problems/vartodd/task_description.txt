Your global task is mutating a Python tuner that minimizes T-count by tuning a FastTODD policy. 
Objective: discover policies that reach the lowest decomposition rank (T-count) more reliably across seeds. Current SOTA is 397

### TODD
Decomposition is represented by a matrix. Rank equals number of columns and equals T-count.
Every step it is action generation and choice according to policy for current rank. Every action lead to new matrix with lesser ranks (num of rows)
There are two stages: TOHPE (reseach greatly restricted space of action and easily find z with best reduction) and TODD (much heavier exploration leads to time increasing). 

### FastTODD Policy Variables:
1) Search policy variables:
 - min_z_to_research:int — determine main complexity of action generation during TODD search 
 - beamsearch_width:int — after every iteration keeps beamsearch_width best matrix according to final_weights. Increse runtime in beamsearch_width times. 
 - todd_width:int — how many actions returned by every iterations (if 1 there is no beamsearch) 
 - gen_part:float in [0,1] — How many of all generators of nullspace check
 - num_samples:int — Number of samples evaluated per candidate z. Each z defines a binary vector space of dimension dim (up to 2^dim possible assignments). If num_samples >= 2**dim, evaluate the full space (i.e., use only 2**dim samples). 
 - min_pool_size:int — number of minimal actions should be found during exploration search
 - max_pool_size:int — number of best kept actions during exploration search
 - tohpe_num_best:int — how many z will be obtained from TOHPE
 - try_only_tohpe:bool — if True and num of actions found during TOHPE stage more than min_pool_size than dont use TODD search. In other cases run TODD despite True.
 - max_tohpe:int — number of action choosen during TOHPE search
 - max_from_single_ns:int — limits num of choosen actions from one z
 - max_reduction, min_reduction:int — clamp allowed reduction for rank, if not such actions push other actions in pool
 - temperature:float in [0, 1] — for softmax final choice based on final score

2) Choice policy variables:
 - pool_weights:ExplorationScore(weights=[wred, wdim, wbucket, wvw, wz],centers=[cred, cdim, cbucket, cvw, cz], pow=pow) — used during exploration with TOHPE and TODD 
 - final_weights:FinalizationScore(weights=[wred, wdim, wbucket, wvw, wz, wtohpe],centers=[cred, cdim, cbucket, cvw, cz, wtohpe], pow=pow) — used during final choice among best actions from exploration stage
  *red — reduction of rank from given action
  *dim — the size of space from which action were sampled (num_samples<=2**dim)
  *bucket — the early estimiation in reduction for given z
  *vw — how great is the changing in matix for given action
  *z — number of ones in z bool vector
  *tohpe — the size of future TOHPE space dim !! Can be very time consuming if max_pool_size is big: wrothpe=0 leads to faster evalutions
  *pow - can be negative
  score = \sum_i w_i |x_i - c_i|^pow: where x_i \in [0,1] - normalized parameters described above.
  *default vectors values are zero, using all the parameters can be too greedy.

### Rank Scheduling
All parameters may be represented as:
- a constant value, or
- a rank schedule, e.g. two lists: rank_thresholds (sorted from high to low), values. The choice of value is as follows:
for i in range(0, len(rank_thresholds)):
    if rank_thresholds[i] <= current_rank:
        return values[i]
return values[-1]


### Evaluator Interface 
-evaluator.policy_mapping() — set up parameters for todd search (Must be implemented)
  *use evaluator.map_par(fun=<callable>, rank_thr=<int>) — The only way get variable. Map parameter via fun and check threshold limits. Notes: every call move return fun(hidden_parameters[cur_idx]) and cur_idx++ if rank of init matrix > rank_thr, else do nothing (return zero)
  *evaluator.set_X(constant) — set up constant shcedule, where X is one of policy variables
  *evaluator.set_X(ranks: List[int], values: List[Any]) — set up rank schedule, where X is one of policy variables
-evaluator(params) — Evalute loss for given parameters (Must be implemented)
  *evaluator.run(params, seeds): List[int] — Run search and return best find Rank for every seed
-evaluator.best_pathes — keep best overall pathes (list[list[Matrix]]) found during all search
-evaluator.best_rank — return current best seen rank
-evaluator.path_num — num of current best pathes which can be chosen for reinitialization. Can be changed after run with reinitialization.
-evaluator.set_up_new_init(path_num=<int>, rank_thr=<int>, xopt=<List[float]>):List[float]|None — path_num and rank_thr define next init matrix for todd: set last matrix with rank > rank_thr for path with num best_num. Return None if all matrix > rank_thr else active parameters 
-evaluator.extract_active() — return current active parameters (e.g. parameters in the limits of possible rank thresholds)
-get_matrix() — required to obtain initial matrix
