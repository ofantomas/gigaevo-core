TASK DEFINITION -- HEILBRONN TRIANGLE PROBLEM (ADVERSARIAL CONSTRUCTOR)

You are a CONSTRUCTOR (Generator) in a two-population adversarial co-evolution game, analogous to a GAN:
- YOU (Pop A, the "Generator"): produce point configurations
- YOUR OPPONENTS (Pop B, the "Improver"): evolve perturbation operators that take your configuration and try to improve it (increase its min_area)

THE GAME
Your opponents are other evolving programs. They receive your (11,2) point array and return a modified version. As the Improver population evolves, a configuration that resists simple perturbations today will be cracked by more powerful methods tomorrow.

You are evaluated every turn against the current top-K Improvers and tracked over your lifetime. Two things determine your standing:

1) FITNESS — the intrinsic `min_area` of your configuration, normalized to [0, 1] as `min(min_area / 0.0365, 1.0)`. Higher is better.
2) WINS — how many distinct Improvers have already failed against you (i.e. did not strictly increase your min_area). This number accumulates across the Hall-of-Fame rotations you have survived.

A beautiful configuration that every Improver can crack is worse than a slightly-flatter configuration sitting at a deep, well-isolated local optimum that no Improver has managed to improve. See the metric definitions in your prompt for the exact formulas and units.

WHAT WINS THIS GAME
- Configurations at genuine local optima of the min_area landscape — where no single-point perturbation raises min_area
- Deep basins beat sharp spikes: the global optimum (min_area ≈ 0.0365) is perfectly un-improvable, but a slightly-lower configuration surrounded by a wide flat plateau can be just as hard for Improvers to crack
- Rare, exotic symmetries (odd rotation angles, boundary-hugging layouts) often accumulate `wins` faster than textbook near-optimal grids, because they force Improvers to invent new strategies

OBJECTIVE
Return 11 distinct 2D coordinates inside a unit-area equilateral triangle such that:
1) (Containment) All points lie inside or on the boundary of the enclosing triangle
2) (Distinctness) All points are distinct
3) (Non-degeneracy) No three points are collinear
4) (Fitness) The minimum area among all C(11,3) = 165 triangles is maximized
5) (Hard to improve) The configuration sits in a deep basin — no small perturbation increases min_area

Output: (11, 2) NumPy array  |  Target: min_triangle_area ≈ 0.0365

CONSTRAINTS
- Enclosing triangle: equilateral, area = 1.0, flat-bottomed
  Vertices: A=(0,0), B=(1.5197,0), C=(0.7598,1.3161)
- Exactly 11 distinct points, all within or on boundary
- No collinear triplets

HELPER FUNCTIONS (must be imported from helper.py — not available globally):
  from helper import get_unit_triangle, get_smallest_triangle_area, is_inside_triangle

- `get_unit_triangle() -> (A, B, C)`
    Returns 3 arrays each shape (2,) — vertices of the unit-area equilateral triangle.

- `get_smallest_triangle_area(coordinates: np.ndarray) -> float`
    Input:  (n, 2) float array of n points
    Output: minimum area among all C(n,3) triangles formed by the points

- `is_inside_triangle(points, a, b, c) -> bool`
    Input:  points — shape (n, 2) or (2,) for a single point; a, b, c — each shape (2,) vertices
    Output: True if ALL given points lie inside or on the triangle boundary

OUTPUT FORMAT
Implement `def entrypoint():` that returns a (11, 2) NumPy array.
Use numpy, scipy, sklearn, or standard library as needed.
Fix random seeds if using randomness.
