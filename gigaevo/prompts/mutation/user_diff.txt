## SEARCH/REPLACE FORMAT (STRICT)

Emit your change as one or more SEARCH/REPLACE blocks in the `code` field. The blocks are applied in order to Parent 1's source via literal string replacement.

Each block has exactly this shape:

<<<<<<< SEARCH
<verbatim block from Parent 1, copied byte-for-byte including indentation>
=======
<the replacement block, or empty to delete the SEARCH block>
>>>>>>> REPLACE

Rules enforced by the runtime:

- Verbatim SEARCH: the text between `<<<<<<< SEARCH` and `=======` must appear EXACTLY ONCE in Parent 1's current source. Copy whitespace, indentation, blank lines, and trailing characters byte-for-byte.
- Uniqueness: if the snippet appears in several places, expand SEARCH upward/downward with surrounding lines until the match is unique.
- Markers literal: `<<<<<<< SEARCH`, `=======`, and `>>>>>>> REPLACE` must each appear on their own line.
- Multiple edits: emit multiple consecutive blocks. They are applied in order; each operates on the result of the previous one.
- Replace whole logical units when possible, such as a full function, branch, or block.
- No outer wrapping: the `code` field starts directly with `<<<<<<< SEARCH`. Do not wrap blocks in markdown fences.
- No full source: do not emit the entire mutated program.
- The patched program must be valid Python.
- Diff mode has exactly one parent. All SEARCH blocks must target Parent 1 only.

Tiny example. Parent 1 contains:

def score(x):
    return x ** 2

def main():
    print(score(3))

Acceptable `code` field to change the exponent to 3:

<<<<<<< SEARCH
def score(x):
    return x ** 2
=======
def score(x):
    return x ** 3
>>>>>>> REPLACE
