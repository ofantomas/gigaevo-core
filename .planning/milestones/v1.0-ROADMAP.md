# Milestone v1.0: GigaEvo Monitoring & Tools Overhaul

**Status:** SHIPPED 2026-04-12
**Phases:** 1-5
**Total Plans:** 19

## Overview

Bottom-up build of GigaEvo's monitoring, notification, and CLI tooling. Shared library first, then notification channels, then watchdog engine with plugins, then CLI shell, then composite commands and migration. New code in `gigaevo/monitoring/` and `gigaevo/cli/`.

## Phases

### Phase 1: Foundation (3/3 plans)
**Goal**: Shared monitoring library — RunSpec parser, RunSnapshot, ExperimentMonitor, AlertDetector, ExperimentManifest.
**Plans**: 01-01 (RunSpec + RunSnapshot + Redis queries), 01-02 (ExperimentManifest Pydantic schema), 01-03 (AlertDetector multi-signal stall detection)
**Tests**: 135

### Phase 2: Notifications (4/4 plans)
**Goal**: Dual-channel notification delivery — Telegram + PR comments with strategy pattern.
**Plans**: 02-01 (NotificationChannel ABC + StatusUpdate + formatters), 02-02 (TelegramChannel with httpx + retry), 02-03 (GitHubPRChannel with plot upload), 02-04 (NotificationDispatcher fan-out + cooldown)
**Tests**: 251 (cumulative)

### Phase 3: Watchdog (5/5 plans)
**Goal**: Generic watchdog engine with plugin system for all experiment types.
**Plans**: 03-01 (WatchdogPlugin ABC + registry), 03-02 (WatchdogEngine core loop), 03-03 (SoloPlugin + AdversarialPlugin), 03-04 (HeilbronPlugin + PromptCoevoPlugin), 03-05 (Template shim + integration test)
**Tests**: 339 (cumulative)

### Phase 4: CLI (4/4 plans)
**Goal**: Unified `gigaevo` entry point with LazyGroup, OutputFormatter, RunResolver.
**Plans**: 04-01 (CLI skeleton), 04-02 (status, trajectory, top, logs), 04-03 (plot + export subcommands), 04-04 (flush + watchdog)
**Tests**: 87 CLI tests

### Phase 5: Integration (3/3 plans)
**Goal**: Composite lifecycle commands, tool absorption, docs.
**Plans**: 05-01 (checkpoint command), 05-02 (launch/closeout/restart), 05-03 (docs + CLI reference)
**Tests**: 96 CLI tests (cumulative)

## Milestone Summary

**Key Decisions:**
- Click 8.x (not Typer) — already in codebase
- Rich 14.x for output — tables, JSON, progress, panels
- httpx for Telegram — already a dependency
- Dict registry (not entry_points) — only 3-4 experiment types
- Manifest is OPTIONAL — --run mode is first-class
- Replace tools, not wrap — clean break from inconsistent APIs

**Stats:**
- 13,125 lines of Python (monitoring + CLI)
- 435 tests (96 CLI + 339 monitoring)
- 145 files changed, 22,468 insertions
