# Requirements Archive: v1.0 — GigaEvo Monitoring & Tools Overhaul

**Archived:** 2026-04-12
**Original:** .planning/REQUIREMENTS.md (2026-04-11)

## v1 Requirements — All Complete

### CLI
- [x] **CLI-01**: Unified `gigaevo` entry point with LazyGroup subcommand hierarchy
- [x] **CLI-02**: Consistent global flags: `--experiment`, `--run`, `--quiet`, `--verbose`
- [x] **CLI-03**: Both modes: `--experiment` (auto-discovers) and `--run` (ad-hoc)
- [x] **CLI-04**: Structured output via `--format {table,json,csv,markdown}` with pipe auto-detection
- [x] **CLI-05**: Destructive commands require `--confirm`, support `--dry-run`
- [x] **CLI-06**: Click 8.x + LazyGroup for lazy imports

### Manifest
- [x] **MAN-01**: Strict Pydantic-validated ExperimentManifest schema
- [x] **MAN-02**: Manifest is OPTIONAL — all tools work via `--run` mode
- [x] **MAN-03**: Schema exported as JSON Schema

### Monitoring
- [x] **MON-01**: Generic WatchdogEngine with WatchdogPlugin ABC
- [x] **MON-02**: Plugin system with registry, generate_plots(), format_status_body()
- [x] **MON-03**: Built-in plugins: solo, adversarial, heilbron, prompt-coevo
- [x] **MON-04**: PID liveness, heartbeat TTL, stale/stall detection
- [x] **MON-05**: Gen-by-gen trajectory tracking via ExperimentMonitor
- [x] **MON-06**: Stagnation detection in AlertDetector
- [x] **MON-07**: `gigaevo logs <label>` with `--follow` for live tailing

### Notifications
- [x] **NOT-01**: Telegram delivery via httpx with retry and startup probe
- [x] **NOT-02**: Full status table in Telegram messages
- [x] **NOT-03**: Fitness curve plots as Telegram photos
- [x] **NOT-04**: NotificationChannel ABC + NotificationDispatcher fan-out
- [x] **NOT-05**: PR comments with structured markdown tables and plot links
- [x] **NOT-06**: Both channels get identical data
- [x] **NOT-07**: Alert types with severity and cooldown

### Plotting
- [x] **PLT-01**: Comparison curves with rolling fitness, smoothing, confidence bands
- [x] **PLT-02**: Trajectory plots with frontier/mean fitness and baselines
- [x] **PLT-03**: CSV export (evolution data + frontier-only)

### Shared Library
- [x] **LIB-01**: `gigaevo/monitoring/` package with RunSnapshot, plot rendering, formatters
- [x] **LIB-02**: Unified RunSpec parser replacing 3 divergent implementations
- [x] **LIB-03**: Consistent error handling via loguru with contextual prefixes

### Extras
- [x] **EXT-01**: `gigaevo checkpoint` composite command
- [x] **EXT-02**: Pluggable anomaly detector (AlertDetector with multi-signal rules)
- [x] **EXT-03**: `gigaevo launch` composite with --confirm gate
- [x] **EXT-04**: `gigaevo closeout` composite with --confirm gate
- [x] **EXT-05**: `gigaevo restart` composite with --confirm gate

### Migration
- [x] **MIG-01**: New code in `gigaevo/monitoring/` and `gigaevo/cli/` only
- [x] **MIG-02**: Old tools preserved (running experiment constraint); CLI docs reference both
- [x] **MIG-03**: httpx used for Telegram (requests not yet removed — running experiment constraint)
