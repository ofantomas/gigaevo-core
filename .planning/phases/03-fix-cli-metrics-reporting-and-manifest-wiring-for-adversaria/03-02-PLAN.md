```python
def _format_metric_value(value: float | None, name: str, specs: dict[str, dict]) -> str:
    """Format a metric value according to its metrics.yaml spec.

    - sentinel_value: display as "N/A"
    - upper_bound == 1.0: display as percentage (value * 100, fewer decimals)
    - decimals: control decimal places
    - Default: 3 decimal places
    """
    if value is None:
        return "?"
    spec = specs.get(name, {})
    sentinel = spec.get("sentinel_value")
    if sentinel is not None and value == sentinel:
        return "N/A"
    decimals = spec.get("decimals", 3)
    upper_bound = spec.get("upper_bound")
    if upper_bound is not None and upper_bound == 1.0:
        # Display as percentage: 0.85 -> "85.000%"
        pct_decimals = max(0, decimals - 2)
        return f"{value * 100:.{pct_decimals}f}%"
    return f"{value:.{decimals}f}"
```

**However**, `RunConfig` does not carry `problem_name`. The simplest approach that avoids modifying `RunConfig` (which would be a larger change): accept `metric_specs` as an optional parameter to `_snapshot_to_row`, and load specs at the command level when in --experiment mode.

**Update the `status` command to load specs:**

In the `status()` function, after resolving run_configs but before collecting snapshots:

```python
# Load metric display specs from experiment manifest
metric_specs: dict[str, dict] = {}
if experiment:
    try:
        from gigaevo.cli.run_resolver import _load_manifest
        from pathlib import Path
        import yaml
        manifest = _load_manifest(experiment)
        seen: set[str] = set()
        for run in manifest.runs:
            if run.problem_name not in seen:
                seen.add(run.problem_name)
                path = Path("problems") / run.problem_name / "metrics.yaml"
                if path.exists():
                    with open(path) as f:
                        data = yaml.safe_load(f)
                    if isinstance(data, dict):
                        metric_specs.update(data.get("specs", {}))
    except Exception:
        pass  # fall back to raw display
```

Then modify `_snapshot_to_row` to accept and use specs:

```python
def _snapshot_to_row(snapshot: RunSnapshot, metric_specs: dict[str, dict] | None = None) -> dict:

        "Label": snapshot.run_spec.label,

        "Gen": snapshot.generation,

    specs = metric_specs or {}
    for name, value in snapshot.metrics.items():
        col_name = name.replace("_", " ").title()
        row[col_name] = _format_metric_value(value, name, specs)
```

**Note on backward compatibility:** The analyze and collect commands currently use `--prefix`/`--db` as required options, not the standard `--experiment`/`--run` flags. Changing their CLI interface is a larger refactor that would break any existing scripts or skills using them. For this phase, we only wire them into the registry so they are discoverable via `gigaevo analyze` and `gigaevo collect`. The `--prefix`/`--db` flags continue to work because they are defined on the commands themselves, and the global `--experiment`/`--run` flags from the main group are available via `ctx.obj` but not required by these commands.

Do NOT modify the analyze.py or collect.py function signatures or required options in this task. Only add the registry entries.
