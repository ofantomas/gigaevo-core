<objective>
Fix metric discovery propagation in watchdog_cmd, trajectory, and top commands, and add watchdog_plugin field to manifest schemas.

Purpose: The v1.0 monitoring infrastructure correctly loads metrics from metrics.yaml via RunResolver, but watchdog_cmd bypasses this (hardcoding ["fitness"]), trajectory/top accept only a single --metric flag with no auto-discovery, and the manifest schema lacks a watchdog_plugin field needed for prompt_coevo experiments. This plan fixes bugs 1, 2, 3, and 5 from the research.

Output: Four CLI/schema files fixed with proper metric propagation, five test files extended with multi-metric and adversarial coverage.
</objective>

**Bug 1 fix -- watchdog_cmd.py:**

In `watchdog_cmd.py`, replace the RunConfig construction loop (lines 76-80) to load metric_names from metrics.yaml for each run:

```python
from gigaevo.cli.run_resolver import _load_metric_names
```

Change the loop body from:
```python
for run in manifest.runs:
    spec = RunSpec(prefix=run.prefix, db=run.db, label=run.label)
    rc = RunConfig(run_spec=spec, pid=run.pid)
    run_configs.append(rc)
```
to:
```python
for run in manifest.runs:
    spec = RunSpec(prefix=run.prefix, db=run.db, label=run.label)
    metric_names = _load_metric_names(run.problem_name)
    rc = RunConfig(run_spec=spec, metric_names=metric_names, pid=run.pid)
    run_configs.append(rc)
```

This matches the existing pattern in `RunResolver._resolve_from_experiment()`.

In the `trajectory()` function body, after resolving run_configs, add auto-discovery logic:

```python
# Auto-discover metrics from run_configs when none explicitly specified
if not metric:
    # Collect unique metric names from all run configs, preserving order
    seen = set()
    discovered = []
    for rc in run_configs:
        for m in rc.metric_names:
            if m not in seen:
                seen.add(m)
                discovered.append(m)
    metrics_to_show = discovered if discovered else ["fitness"]
else:
    metrics_to_show = list(metric)
```

**Bug 3 fix -- top.py:**

After resolving run_configs but before using the `metric` parameter, add manifest-aware default:

```python
# Use manifest's problem.metric_name as default when in --experiment mode

if metric == "fitness" and experiment:
    from tools.experiment.manifest import load_manifest
    try:
        manifest = load_manifest(experiment)
        if manifest.problem.metric_name:
            metric = manifest.problem.metric_name
    except Exception:
        pass  # fall back to "fitness"
```

This must go after the `experiment = ctx.obj["experiment"]` line and before `RunResolver.resolve()`. The condition `metric == "fitness"` detects that the user used the default (did not explicitly pass --metric). When the manifest specifies `metric_name: actual_fitness`, the top command will rank by `actual_fitness` automatically.
