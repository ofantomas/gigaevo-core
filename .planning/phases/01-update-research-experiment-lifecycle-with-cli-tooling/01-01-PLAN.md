```python
class LazyGroup(click.Group):
    """Click group that imports subcommand modules only when invoked."""

@click.group(cls=LazyGroup)
@click.option("-e", "--experiment", ...)
@click.option("-r", "--run", ...)
@click.option("-f", "--format", "format_name", ...)
@click.pass_context
def main(ctx, experiment, run, format_name, quiet, verbose, redis_host, redis_port):
    ctx.ensure_object(dict)
    ctx.obj["formatter"] = OutputFormatter(format_name=format_name, quiet=quiet)
    ctx.obj["experiment"] = experiment
    ctx.obj["runs"] = run
```

  <action>
    Create `gigaevo/cli/manifest_cmd.py` with a Click group `manifest` containing these subcommands (per D-02):

    1. **`get`** — Read any manifest field by dotted path.
       - `@click.command()` with `@click.argument("field")` 
       - Special handling for `runs` (tabular output), `status`, `max_generations`, `stopping_rule`, `servers`
       - For dotted paths (e.g. `launch.watchdog_pid`), traverse the `_raw` dict
       - Uses `ctx.obj["formatter"]` for output (table for `runs`, plain value for scalars)
       - If field not found, exit 1 with "Field not found: {field}"

    2. **`set`** — Write a top-level field. Only supports `status` (delegates to `set_status()`).
       - `@click.command()` with `@click.argument("field")` and `@click.argument("value")`
       - For `status`: calls `set_status(experiment, value)`
       - For other fields: exit 1 with "Use 'manifest update' for arbitrary fields"

    3. **`update`** — Write any field by dotted path via `update_manifest()`.
       - `@click.command()` with `@click.argument("path")` and `@click.argument("value")`
       - Splits path on `.` and traverses/creates nested dicts in the updater lambda
       - Auto-converts value: try int(), then float(), then keep as string. If value is "true"/"false" convert to bool. If value is "null"/"None" convert to None.
       - Calls `update_manifest(experiment, updater_fn)`

    4. **`gate`** — Assert status matches expected, exit 0 or 1.
       - `@click.command()` with `@click.argument("expected_status")`
       - Loads manifest, compares status
       - On match: prints "GATE PASSED: {name} status={status}" + summary (runs count, max_gen), exit 0
       - On mismatch: prints "BLOCKED: status={actual}, expected {expected}", exit 1

    5. **`pr-description`** — Generate and optionally push PR description.
       - `@click.command("pr-description")` with `@click.option("--push/--no-push")`
       - Calls `generate_pr_description(experiment)`, prints result
       - If `--push` and manifest has `pr_number`: runs `gh pr edit {pr_number} --body {description}`
       - Writes to `experiments/{exp}/PR_DESCRIPTION.md`

    All subcommands:

    - Use lazy imports: `from tools.experiment.manifest import ...` inside function bodies
    - Follow the same pattern as `gigaevo/cli/lifecycle.py` for getting experiment from context

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> manifest functions | User-provided field paths and values cross into YAML file writes |
| manifest update -> experiment.yaml | Arbitrary dotted paths could overwrite critical fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | manifest update | mitigate | `set_status` uses state machine validation; `update` writes through `update_manifest()` which holds Redis lock and validates schema |
| T-01-02 | Information Disclosure | manifest get | accept | experiment.yaml is already readable by anyone with repo access; no secrets stored in manifest |
| T-01-03 | Denial of Service | manifest gate | accept | gate is a read-only status check with no side effects |
</threat_model>
